# DesktopTodo 桌面待办应用 - 需求文档

> 版本: v1.0
> 目标: 打造一款简洁、高效、沉浸的 macOS 原生桌面待办应用，对标滴答清单核心体验，聚焦个人任务管理场景。

---

## 1. 产品定位

**一句话描述**: 一款轻量、优雅、深度融入 macOS 生态的桌面待办清单应用。

**目标用户**: 个人用户，需要快速记录和管理日常任务，追求效率工具的原生体验。

**核心原则**:
- 极简: 功能克制，不做 All-in-One，专注待办本身
- 原生: 深度适配 macOS 设计语言与交互习惯
- 沉浸: 减少干扰，让用户专注于任务处理

## 2. 功能需求

### 2.1 任务管理 (核心功能)

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 创建任务 | 输入框快速添加，支持回车提交 | P0 |
| 完成任务 | 点击复选框切换完成/未完成状态 | P0 |
| 删除任务 | 左滑删除或 hover 显示删除按钮 | P0 |
| 编辑任务 | 双击任务文本进入编辑模式 | P0 |
| 任务排序 | 支持拖拽调整任务顺序 | P1 |
| 任务筛选 | 全部 / 进行中 / 已完成 | P1 |
| 批量操作 | 批量完成、批量删除 | P2 |

### 2.2 列表管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 默认列表 | 提供一个默认的「收件箱」列表 | P0 |
| 自定义列表 | 创建多个清单（如工作、学习、购物） | P1 |
| 列表切换 | 侧边栏点击切换不同清单 | P1 |
| 列表设置 | 重命名、删除、图标选择 | P1 |
| 智能列表 | 今天 / 最近7天 / 已完成（自动聚合） | P2 |

### 2.3 任务属性

| 属性 | 描述 | 优先级 |
|------|------|--------|
| 截止日期 | 设置任务到期时间 | P1 |
| 优先级 | 高/中/低，用颜色区分（红/黄/蓝） | P1 |
| 标签 | 自定义标签分类，支持多标签 | P2 |
| 备注 | 任务详情描述，支持多行文本 | P2 |
| 子任务 | 一个主任务下可拆分多个子步骤 | P2 |

### 2.4 提醒与通知

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 到期提醒 | 任务截止时发送系统通知 | P1 |
| 定时提醒 | 自定义提醒时间 | P2 |
| 重复任务 | 每天/每周/每月重复 | P2 |

### 2.5 数据与同步

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 本地存储 | 数据持久化到本地 JSON/SQLite | P0 |
| 导入导出 | 支持 JSON/CSV 格式备份与恢复 | P2 |
| iCloud 同步 | 多设备数据同步（远期） | P3 |

## 3. 界面设计需求

### 3.1 整体布局

```
+------------------------------------------+
|  [侧边栏]        |  [主内容区]             |
|                  |                        |
|  📥 收件箱        |  +------------------+  |
|  💼 工作          |  | 输入框: 添加任务... |  |
|  📚 学习          |  +------------------+  |
|  🛒 购物          |                        |
|                  |  ○ 完成设计稿        🔴  |
|  ---             |  ○ 准备周会材料      🟡  |
|  📅 今天          |  ✓ 回复邮件          🔵  |
|  📆 最近7天       |                        |
|  ✅ 已完成        |                        |

```

### 3.2 视觉风格

- **设计语言**: 遵循 macOS Sonoma/Sequel 风格，毛玻璃效果、圆角、柔和阴影
- **配色**: 跟随系统深浅色模式，提供一套清新配色（蓝/绿/紫主题可选）
- **字体**: 系统默认字体（SF Pro），标题 16pt，正文 13pt
- **动画**: 添加/完成/删除任务时有流畅的微动画（弹簧动画）

### 3.3 窗口行为

| 行为 | 描述 | 优先级 |
|------|------|--------|
| 主窗口 | 标准窗口，可调整大小，记住上次位置 | P0 |
| 最小化 | 支持最小化到 Dock | P0 |
| 状态栏模式 | 可选常驻菜单栏，点击弹出迷你窗口 | P1 |
| 全局快捷键 | 自定义快捷键快速唤出（如 Cmd+Shift+T） | P1 |
| 开机启动 | 可选登录时自动启动 | P2 |

## 5. 技术需求

### 5.1 技术栈

- **语言**: Swift 6
- **UI 框架**: SwiftUI（macOS 14+）
- **数据持久化**: SwiftData（苹果现代数据框架，声明式 API，与 SwiftUI 深度集成）
- **通知**: UserNotifications 框架
- **全局快捷键**: Carbon 或 HotKey 库
- **状态栏**: AppKit NSStatusBar

> **选型说明**: 使用 SwiftData 而非 Core Data。SwiftData 是苹果在 WWDC23 推出的现代数据持久化框架，构建在 Core Data 之上但提供更简洁的声明式 API，与 SwiftUI 无缝集成，自动处理数据同步和简单迁移。对于待办事项这种轻量级数据场景，SwiftData 的开发体验更优。无需 iCloud 同步，数据完全本地存储。

### 5.3 数据模型

```swift
// 任务模型
@Model
class TodoItem {
    @Attribute(.unique) var id: UUID
    var title: String
    var isCompleted: Bool
    var createdAt: Date
    var dueDate: Date?
    var priority: Priority
    var notes: String?
    var order: Int
    
    @Relationship(inverse: \TodoList.items)
    var list: TodoList?
    
    @Relationship(inverse: \Tag.items)
    var tags: [Tag]?
}

enum Priority: Int, Codable {
    case none = 0
    case low = 1    // 蓝色
    case medium = 2 // 黄色
    case high = 3   // 红色
}

// 清单模型
@Model
class TodoList {
    @Attribute(.unique) var id: UUID
    var name: String
    var icon: String
    var color: String
    var order: Int
    var createdAt: Date
    
    @Relationship(deleteRule: .cascade)
    var items: [TodoItem]?
}
```

### 9.3 已决策事项

- [x] **SwiftData vs Core Data**: 选择 SwiftData。原因：声明式 API 更简洁、与 SwiftUI 深度集成、自动处理数据同步和简单迁移，适合轻量级待办场景。
- [x] **iCloud 同步**: 不支持。数据完全本地存储，无需考虑跨设备同步、冲突解决等复杂度。
- [x] **开源**: 采用 MIT 许可证开源，方便社区贡献和学习参考。
